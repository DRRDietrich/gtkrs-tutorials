Getting Started

Dependencies

Before we begin, ensure that you have the necessary development files installed for GTK. On Debian platforms, you will install libgtk-3-dev for GTK3, or libgtk-4-dev for GTK4. If you intend to be integrating a GTK WebView into your application, you also need libwebkit2gtk-4.0-dev. If you want to integrate a GTK SourceView, libgtksourceview-4-dev.

On the Rust side of things, you should install cargo-edit with cargo install cargo-edit, because that'll make adding dependencies to your project easier. You should also be using rust-analyzer in your IDE so that you'll have instant feedback about warnings and errors as you type, rather than waiting to find out you made mistakes until you compile. It also has the convenient feature of showing types of variables, and displaying input parameter names in functions you're calling.

Creating Your Project

Create your project

cargo new first-gtk-app
cd first-gtk-app

Add the crates that we'll be using. You'll be using these in all of your GTK apps

cargo add gtk glib async-channel futures-lite

Then open your project in, say, VS Code

code .

Initializing GTK

Lets start by setting your application's name, and initializing GTK.

use std::process;

fn main() {
    glib::set_program_name("First GTK App".into());
    glib::set_application_name("First GTK App");

    if gtk::init().is_err() {
        eprintln!("failed to initialize GTK Application");
        process::exit(1);
    }

    // Thread will block here until the application is quit
    gtk::main();
}
